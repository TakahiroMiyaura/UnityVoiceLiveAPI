# Contributing

[![日本語](https://img.shields.io/badge/lang-日本語-red.svg)](CONTRIBUTING_JP.md)

Thank you for your interest in contributing to VoiceLive API for Unity.

## How to Contribute

### Reporting Issues

Bug reports and feature requests are accepted via [GitHub Issues](https://github.com/TakahiroMiyaura/UnityVoiceLiveAPI/issues).

When creating an issue, please include the following information:

- **For bug reports**
  - Detailed description of the problem
  - Steps to reproduce
  - Expected behavior vs. actual behavior
  - Unity version
  - Target platform
  - Related error messages and logs

- **For feature requests**
  - Description of the proposed feature
  - Use cases and background
  - Implementation suggestions (if any)

### Pull Requests

1. Fork this repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Create a Pull Request

#### Pull Request Guidelines

- Each PR should contain only one feature or fix
- Follow the existing code style
- Update documentation as needed
- Ensure tests pass if applicable

## Development Environment Setup

### Prerequisites

- Unity 6000.0 or later
- .NET SDK (.NET Standard 2.1 compatible)
- Git

### Build Steps

1. Clone the repository
   ```bash
   git clone https://github.com/TakahiroMiyaura/UnityVoiceLiveAPI.git
   ```

2. Open the `Unity/UnityVoiceLiveAPI` project in Unity

3. Required packages will be installed automatically

## Coding Conventions

- Follow C# coding conventions
- Add XML documentation comments to public members
- Use meaningful variable and method names
